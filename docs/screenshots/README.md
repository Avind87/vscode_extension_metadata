# Screenshots Directory

This directory should contain screenshots of the Data Vault 2.1 Metadata Prep App for use in the main README.md.

## Required Screenshots

1. **main-interface.png** - Main interface showing:
   - Table navigation
   - Statistics counter (Hub Hashkeys, Link Hashkeys, Satellites, Last Change)
   - Table sections with hashkey management
   - Satellite configuration

2. **business-concepts.png** - Business concept selection showing:
   - Dropdown with business concepts
   - Auto-generated hashkey name display
   - "Add New Concept" option

3. **hub-hashkeys.png** - Hub hashkey management showing:
   - Multiple hashkey groups
   - Column tags with remove buttons
   - Drag-and-drop handles
   - Up/down buttons for reordering

4. **link-hashkeys.png** - Link hashkey creation showing:
   - Hub hashkey selectors
   - Auto-generated link name
   - Referenced hashkeys display

5. **satellites.png** - Satellite configuration showing:
   - Business concept selector
   - Auto-selected hashkey (read-only)
   - Satellite name (auto-generated, editable)
   - Column selection mode (Select All / Select Explicit)
   - Column inclusion/exclusion lists

6. **global-hashkey-view.png** - Global hashkey view showing:
   - Table of all hashkeys across all tables
   - Business concept column
   - Main column names (editable)
   - Sources list
   - "Apply to All Sources" button

## How to Take Screenshots

1. Open the extension in VS Code (F5)
2. Load a sample DuckDB database
3. Configure some hashkeys, links, and satellites
4. Use your system's screenshot tool to capture the interface
5. Save images as PNG files with the names listed above
6. Ensure images are clear and readable (recommended: 1920x1080 or higher resolution)
